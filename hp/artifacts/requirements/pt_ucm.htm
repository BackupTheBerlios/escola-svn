Modelo de Casos de Uso

 Caso de uso é um mecanismo utilizado para modelagem (registro) de problemas ou sistemas de software. Um caso de uso é uma seqüência de ações que o sistema realiza que produz um resultado observável por parte do usuário. Este resultado deve possuir valor para o usuário. Casos de uso permitem expressar requisitos de forma fácil de ser compreendida por usuários, clientes e desenvolvedores.
Ferramenta: Rational® Rose™ / Microsoft® Word™

Especificações de casos de uso 

As especificações de caso de uso foram criadas usando a ferramenta Rational RequisitePro(r). O número de casos de uso é próximo de 100 e, neste caso, por simplicidade, eles foram agrupados em documentos contendo múltiplas especificações de casos de uso. Cada documento contém apenas casos de uso de "propósitos similares" com o intuito de facilitar a localização deles. 

  • Administração de Programa (admprg)
  • Administrador (admsys)
  • Calendário (calendar)
  • Cadastro de Cargos e Pessoas Físicas/Jurídicas (ccpfj)
  • Controle de Acesso
  • Discente
  • Disciplina
  • Docente
  • Documentos
  • Informes
  • Processo de Seleção
  • Manutenção de Programa de Pós-Graduação
  • Requerimento
 

 

Nota:

Alguns elementos em arquivos de modelos do Rose estão associados a outros documentos. Para permitir a "instalação" do ambiente de desenvolvimento em outros diretórios e máquinas, é necessário estabelecer um diretório principal onde todos os documentos serão depositados em relação a este diretório. Para tal, siga os seguintes passos no Rose:

  • Abra o Rose, selecione File e então Edit Path Map
  • No campo "Symbol", digite SISPG_HOME (todas as letras maiúsculas)
  • No campo "Actual Path", digite o diretório de referência (principal)
  • Selecione Add e então Close
