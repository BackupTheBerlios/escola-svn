Especificação de Casos de Uso

 

Preâmbulo

O SISPG só poderá oferecer acesso a alguns serviços mediante comprovação de autorização por parte do interessado. Este documento contém casos de uso pertinente ao controle de acesso ao sistema. Este controle é baseado na noção de conta. Conta é o mecanismo que permite uma pessoa física ter acesso ao sistema conforme a categoria identificada na conta.

 

 

1.                  UC26 Manutenir conta 

O sistema permitirá o seu uso apenas por pessoas físicas devidamente autorizadas. Esta autorização ocorre quando uma conta é criada para a pessoa física em questão. Associada a cada conta está a pessoa física pertinente e o tipo de categoria correspondente. Um período de validade da conta também deverá ser estabelecido. A categoria define o tipo de privilégio que o novo usuário terá ao interagir com o sistema. Este caso de uso contempla a criação, remoção, consulta e edição de contas.

 

2.                  UC27 Manutenir senha 

Após criada uma conta, a senha pertinente será válida apenas para se ter acesso a outro serviço: Cadastrar senha. Dessa forma, todo usuário receberá uma senha temporária através da qual apenas o serviço que permite trocar esta senha estará disponível. Em conseqüência, assegura-se que ninguém mais, exceto o usuário, poderá divulgar a senha pertinente a sua própria conta. Adicionalmente, este caso de uso deverá ser utilizado para fornecimento de nova senha quando a anterior estiver expirada.

 

3.                  UC79 Validar usuário 

Qualquer ser humano que desejar usufruir os serviços oferecidos pelo SISPG deverá ser devidamente identificado. Conforme a identificação, o sistema oferece o conjunto de serviços disponíveis. Toda interação com o sistema inicia, necessariamente, por segurança, por este caso de uso. Nesta ocasião o sistema poderá, por exemplo, negar o acesso a usuário ou toda uma categoria de usuários, pelos mais variados motivos.

 

3.1               Fluxo de eventos

 

·         O sistema exibe a página inicial. Esta página convida o usuário a se identificar.

·         Usuário fornece sua identificação.

·         Usuário requisita validação.

·         O caso de uso termina.

 

3.2               Fluxos alternativos

 

3.2.1          Identificação inválida

O nome de identificação do usuário e/ou senha estão incorretos. Mensagem será exibida informando esta situação. Os campos utilizados para as informações são limpados. É aguardada nova tentativa de identificação. Após três tentativas inválidas, o sistema estará inoperante, para o usuário em questão, por um período de 60 segundos.

 

3.2.2          Usuário já está usando o sistema

O usuário está executando este processo pela segunda vez, enquanto a primeira tentativa foi satisfatória e o uso do sistema não foi finalizado. Mensagem é exibida informando esta situação. O caso de uso termina.

 

3.2.3          Usuário ou categoria está bloqueado(a)

Quando o acesso de usuário ou categoria do mesmo estiver bloqueado, o sistema informará o fato. O caso de uso é finalizado.

 

3.2.4          Conta expirada

Todo usuário precisa, em intervalo de tempo estipulado pelo administrador, ser recadastrado. Quando se expira a validade de um cadastro de usuário, o acesso correspondente torna-se indisponível. Quando este tentar fazer uso do sistema será avisado de tal fato. O sistema ainda irá sugerir as ações a serem tomadas para que o recadastramento possa ser efetuado. O caso de uso é terminado.
