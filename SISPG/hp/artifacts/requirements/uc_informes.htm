Especificação de Casos de Uso

 

Preâmbulo

Este documento fornece a especificação de vários casos de uso associados a informes.

 

1.                  UC50 Consultar informes 

Usuários e navegantes têm acesso a um variado conjunto de informes, classificados conforme a taxonomia empregada pelo remetente. Não apenas os mais recentes, mas também aqueles já expirados, poderão ser consultados. O acesso a um dado informe depende da categoria para a qual  o informe foi direcionado. Este caso de uso descreve o acesso aos informes disponíveis para uma dada categoria.

 

1.1               Fluxo de eventos

 

·         Identificação de usuário é validada ou se trata de navegente dá início ao presente caso de uso.

·         O sistema apresenta a lista de informes disponíveis conforme a identificação do usuário (ou navegante), classificados do mais recente para os mais antigos.

·         Ocorre consulta aos informes (navegação entre eles).

·         Caso de uso termina quando usuário seleciona outra tarefa.

 

1.2               Fluxos alternativos

 

1.2.1          Usuário seleciona outra classificação de informes

Durante a consulta, o usuário poderá utilizar outra classificação dos informes, diferente da classificação cronológica, apresentada automaticamente. O sistema automaticamente atualizará a lista de informes para refletir a classificação selecionada. Após atualizada, o usuário poderá selecionar outro critério de classificação ou percorrer os informes na nova classificação, entre outras opções.

 

1.2.2          Não existe informes disponíveis

Mensagem informando o fato irá substituir os recursos de navegação pelos informes quando estes estão disponíveis.

 

1.3               Pré-condicação

 

1.3.1          Caso de uso é iniciado apenas com resultado satisfatório do caso de uso Validar Usuário

 

1.4               Pontos de extensão

São várias as extensões de interação. Neste cenário, o usuário, conforme a sua categoria, tem acesso aos informes e à lista de serviços disponíveis.

 

2.                  UC51 Editar informe 

Os usuários que possuem privilégios de categoria para criar informes precisarão, eventualmente, de atualizar o conteúdo de alguns e remover outros.

 

2.1               Fluxo de eventos

 

2.1.1          Início

Usuário clica opção Editar informes.

 

2.1.2          Sistema abre a edição de informes.

Inicia-se o caso de uso Selecionar informe. As opções de edição e remoção de informe estarão habilitadas, disponíveis para o usuário, se e somente se há informe selecionado.

 

2.1.3          Usuário requisita edição de informe selecionado

Inicia-se o caso de uso Alterar informe.

 

2.1.4          Conclusão

Usuário visita outra página, seleciona outra opção disponível, interrompe execução do browser ou volta para a página inicial da sua categoria.

 

2.2               Fluxos alternativos

 

2.2.1          Usuário requisita remoção do informe selecionado

Inicia-se o caso de uso Remover informe.

 

2.2.2          Usuário requisita criação de informe

Inicia-se caso de uso Criar informe.

 

 

2.3               Ponto de extensão

·         Usuário identifica necessidade de criar novo informe e inicia caso de uso Criar informe.

·         Usuário requisita edição de informe, após selecionado e inicia-se Alterar informe.

·         Usuário requisita remoção de informe selecionado, inicia-se Remover informe.

 

3.                  UC52 Criar informe 

Usuário cria informes através da interação descrita neste caso de uso. Por ser similar àquela de Alterar informe, apenas a interação deste último caso de uso será comentada. Convém ressaltar que a alteração de um informe difere da criação apenas nos aspectos abaixo:

·         Ao alterar, todos os campos de informação exibidos contêm as informações do informe que se deseja alterar. Ao criar, os campos estão limpos.

·         Ao confirmar uma alteração, o sistema irá realizar as atualizações necessárias para que os demais usuários possam imediatamente consultá-las. Ao criar, um novo informe será divulgado para a categoria de usuários pertinente.

 

4.                  UC53 Excluir informe 

Após selecionado um informe, o usuário poderá removê-lo, caso possua privilégio suficiente.

 

4.1               Fluxo de eventos

 

4.1.1          Início

Usuário requisita remoção de informe selecionado.

 

4.1.2          Sistema exibe informação de alerta e requisita confirmação

 

4.1.3          Usuário confirma

 

4.2               Fluxos alternativos

 

4.2.1          Usuário não confirma remoção

Sistema retorna à situação anterior, inicia-se Selecionar de informe.

 

5.                  UC54 Limpar base de informes 

Informes "muito antigos" poderão ser desconsiderados de consultas posteriores. Após considerável período de execução do sistema, um grande número deles poderá sobrecarregar a execução de forma desnecessária.
