ANEXO A – ENGENHARIA DE REQUISITOS DO PROGRAMA DE PÓS-GRADUAÇÃO

 

ESPECIFICAÇÃO DOS CASOS DE USO

 

Preâmbulo

 

 

O Programa de Pós-Graduação, tem o foco de Coordenação, Corpo docente, Disciplina, Coordenadoria, Área de Concentração, Atividade, Avaliação Capes. Todo programa de pós-graduação precisa ser identificado para o sistema. Parte das informações pode ser alterada pela coordenação correspondente. Em geral estas informações referem-se a endereços, telefones e outras que sofrem freqüentes alterações. Os casos de uso pertinentes à manutenção de programas são descritos neste documento. As informações pertinentes a um programa são públicas e, em conseqüência, estão disponíveis para todos os usuários.

 

 

1 - Caso de Uso P.P.G.1 – Consultar Programa Pós-Graduação

 ID: P.P.G.1

  

Os programas são compostos de informações de interesse generalizado. Por exemplo, pretendentes a candidatos gostariam de ter acesso ao conteúdo dos programas nos quais desejam ser inscritos. A comunidade acadêmica, em geral, deseja consultar o conteúdo dos Programas de Pós-Graduação. Convém ressaltar que, anualmente é produzido um documento contendo todas as informações de cada um dos programas.

 

1.1 - Fluxo de Eventos

 

     

1.1.1       - Fluxo Básico

 

01- Usuário seleciona Programa de Pós-Graduação;

02- Usuário requisita confirmação da operação;

03- Sistema confirma a operação, listando Programa de Pós-Graduação.

 

 

1.1.2       -  Fluxo Alternativo

 

  • Não possui fluxo alternativo.

 

 

2- Caso de Uso P.P.G.2 – Desativar Programa Pós-Graduação

ID: P.P.G.2

 

                                             

Este caso de uso permite informar ao sistema a interrupção das atividades de um dado programa. Após esta operação, apenas aquelas operações de consulta associadas ao programa serão permitidas. Para executar Desativar Programa, é permitido somente à alguns usuários.

 

2.1 - Fluxo de Eventos:

 

2.1.1 - Fluxo Básico:

 

01- Usuário seleciona um Programa de Pós-Graduação que deseja desativar;

02-  Usuário solicita desativação do programa;

03 - Usuário requisita a confirmação da operação;

04 - Sistema confirma a operação.

 

2.1.2 - Fluxo Alternativo:

 

·        No item 03, caso o Usuário seleciona um Programa de Pós-Graduação que esteja em andamento, o sistema exibe mensagem que o programa selecionado não pode ser desativado.

 

3- Caso de Uso P.P.G.3 – Alterar Programa Pós-Graduação

ID: P.P.G.3

 

Este caso de uso tem como função alterar informações de um Programa de Pós-Graduação.

 

3.1 - Fluxo de Eventos:

 

3.1.1 - Fluxo Básico:

 

            01- Usuário seleciona um Programa de Pós-Graduação;

02- Usuário preenche as informações que deseja alterar;

03- Usuário requisita a confirmação da operação;

04- Sistema confirma a operação, alterando as informações no Programa

de Pós-Graduação.

 

3.1.2 - Fluxo Alternativo:

 

·        No item 3, o sistema exibe mensagem contendo campos obrigatórios que não foram preenchidos e aqueles contendo informações inconsistentes, o sistema sugere que sejam fornecidas e alteradas, respectivamente.
