ANEXO A – ENGENHARIA DE REQUISITOS DOS INFORMES

 

ESPECIFICAÇÃO DOS CASOS DE USO

 

Preâmbulo

 

As coordenações e demais instâncias superiores deverão fornecer informações para toda a comunidade de usuários, ou para categorias específicas. Calendário de programas, palestras e cursos oferecidos estão entre as possíveis informações a serem veiculadas. Em qualquer momento, todos os informes já produzidos, devidamente classificados, poderão ser consultados.

 

1 – Caso de Uso IN.1 – Criar Informe

ID: IN.1

             

Um usuário cria informes através da interação descrita neste caso de uso. Ao criar, o novo informe será exibido para a categoria de usuários destino do informe.

 

Fluxo de Eventos

 

Fluxo Básico:

           

            01-Usuário requisita criação de informe;

02-Se usuário deseja associar anexo, seleciona a criação de um anexo;

            03-Usuário digita texto do informe;

            04-Sistema pede classificação do informe;

            05-Usuário define tipo do informe;

06-Sistema confirma criação do informe e o armazena em uma lista para programação futura.        

 

Fluxo Alternativo:

 

·        Sistema descarta a criação de informe.          

 

 

2 – Caso de Uso IN.2 – Programar Informe

ID: IN.2

 

            Um informe poderá ser programado para ser exibido apenas em um período informado.

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário requisita informes não programados;

02-Sistema exibe lista de informes;

            03-Usuário seleciona o informe;

            04-Usuário define período;

            05-Sistema grava período para exibição.

           

Fluxo Alternativo:

 

·        No item 04, se período for inválido, o sistema exibe uma mensagem exigindo um novo período.

 

 

3 – Caso de Uso IN.3 – Remover Informe

ID: IN.3

 

Após selecionado um informe, o usuário poderá removê-lo, caso possua privilégio suficiente.

 

 

Fluxo de Eventos

 

Fluxo Básico:

           

01-Usuário seleciona o Informe que deseja remover;

02-Sistema exibe informação de alerta e requisita confirmação de remoção;

03-Usuário confirma;

            04-Sistema elimina Informe da lista.

           

Fluxo Alternativo:

                        .

·        No item 03, se o Usuário não confirmar a remoção, sistema exibe mensagem que o Informe não foi eliminado.

 

 

 

4 – Caso de Uso IN.4 – Consultar Informe

ID: IN.4

 

 

O acesso a um informe depende da categoria para a qual o informe foi programado.  Este caso de uso descreve a consulta de informes disponíveis para cada categoria.

 

 

Fluxo de Eventos

 

Fluxo Básico:

           

01-Usuário requisita os informes disponíveis;

02-Sistema exibe lista de informes disponíveis para a categoria de acesso deste usuário;

03-Usuário seleciona informe;

            04-Sistema exibe informe selecionado.

           

 

Fluxo Alternativo:

                        .

·        No item 01, sistema exibe mensagem que não há informes na lista.
