ANEXO A – ENGENHARIA DE REQUISITOS GERIR CORPO SOCIAL

 

ESPECIFICAÇÃO DOS CASOS DE USO

 

Preâmbulo

 

Instituições de ensino envolvem-se em inúmeras interações com pessoas (alunos, professores, pesquisadores, ...) e outras instituições (universidades, fornecedores, ...). No âmbito de uma instituição há muitos cargos e funções cujos ocupantes rotineiramente são alterados. A manutenção das informações atualizadas é imprescindível para o bom andamento de atividades.

 

 

1 – Caso de Uso C.S.1 - Pré-Cadastro de Pessoa Física

ID: C.S.1

 

Descrição:

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário requisita pré-cadastro de Pessoa Física;

02-Sistema exibe formulário de pré-cadastro;

03-Usuário preenche formulário;

04-Sistema verifica validade de dados;

05-Sistema efetua pré-cadastro de Pessoa Física.

 

Fluxo Alternativo:

            

·         No item 4, se os dados forem inválidos ou estiverem incompletos, sistema exibe dados mostrando quais campos estão inválidos ou incompletos e retorna ao item 3 do Fluxo Básico.

 

·         No item 4, caso o Pré-Cadastro já exista, sistema informa que registro já existe.

 

 

 

2 – Caso de Uso C.S.2 - Cadastrar Pessoa Física

ID: C.S.2

 

Descrição:

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário requisita Cadastro de Pessoa Física;

02-Sistema exibe tela de cadastro;

03-Usuário seleciona pré-cadastro de Pessoa Física;

04-Sistema exibe pré-cadastro de Pessoa Física;

05-Usuário confere dados do pré-cadastro de Pessoa Física;

06-Sistema informa que dados foram cadastrados corretamente.

 

            Fluxo Alternativo:

            

·         No item 5, caso os dados do Pré-Cadastro estejam incompletos, o sistema exibe pré-cadastro mostrando quais campos estão incompletos.

ID: C.S.3

 

Descrição:

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário seleciona Pessoa Física;

02-Sistema exibe registros encontrados.

 

            Fluxo Alternativo:

 

·         No item 1, sistema informa que registro não existe.

 

 

4 – Caso de Uso C.S.4 - Editar Pessoa Física

ID: C.S.4

 

Descrição:

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário seleciona Pessoa Física a ser editada;

02-Usuário faz edição de dados;

03-Sistema valida os dados e os disponibiliza de forma editada.

 

            Fluxo Alternativo:

         

·         No item 3, se a validação falhar, sistema não disponibiliza os dados e exibe mensagem de dados inválidos.

 

5 – Caso de Uso C.S.5 - Excluir Pessoa Física

ID: C.S.5

 

Descrição:

 

Fluxo de Eventos

 

Fluxo Básico:

 

01-Usuário seleciona Pessoa Física a ser excluída;

02-Usuário solicita a exclusão da Pessoa Física selecionada;

03-Sistema solicita confirmação de exclusão;

04-Usuário confirma a exclusão;

05-Sistema remove a Pessoa Física.

 

Fluxo Alternativo:

·         No item 3, caso o Usuário não confirme a exclusão, o sistema exibe uma mensagem informando que a Pessoa Física não foi removida.
