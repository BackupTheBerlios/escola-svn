ANEXO A  ENGENHARIA DE REQUISITOS DE CALENDÁRIO

 

ESPECIFICAÇÃO DOS CASOS DE USO

 

Preâmbulo

 

As atividades da pós-graduação serão marcadas por um calendário geral, comum a todos os programas. Periodicamente este calendário será editado (alteração de períodos, datas, ...) para refletir as decisões da pós-graduação (provavelmente uma vez por semestre). Cada programa terá o seu próprio calendário. Este último, contudo, será estabelecido conforme a liberdade oferecida no calendário geral e as próprias peculiaridades. Por exemplo, o calendário da pós-graduação pode estabelecer de 01/01/2005 até 25/01/2005 para a inscrição de candidatos. Neste caso, cada programa fará uso deste período para inscrição ou, caso convenientes, estabelecerá um período menor, mas dentro do período no calendário geral. Invariavelmente no dia 26/01/2005 poder-se-ia computar o total de inscritos em todos os programas. O calendário geral é uma composição de eventos predefinidos de interesse de toda a pós-graduação. Este conjunto não pode ser alterado (acrescido ou reduzido) sem grandes implicações.

 

 

1 - Caso de Uso C.L.1 Cadastrar Calendário

ID: C.L.1

 

A finalidade deste caso de uso é realizar o cadastro de um calendário que será seguido pelos diversos funcionários para a definição de atividades acadêmicas. O calendário pode ser de 2 tipos: calendário geral e calendário de programa.

 

 

Fluxo de Eventos:

 

            Fluxo Básico:

 

01- Usuário preenche o tipo de calendário que deseja cadastrar;

02-  Usuário fornece as informações para cadastro de calendário;

03- Usuário requisita a confirmação da operação;

04-  Sistema confirma a operação, cadastrando o Calendário.

 

Fluxo Alternativo:

 

·        No item 3, o sistema exibe mensagem contendo campos obrigatórios que não foram preenchidos e aqueles contendo informações inconsistentes, o sistema sugere que sejam fornecidas e alteradas, respectivamente.

           

 

2 - Caso de Uso C.L.2 Cadastrar Atividade

ID: C.L.2

 

Este caso de uso tem como finalidade realizar o cadastro de uma determinada atividade, podendo ela estar relacionada à PRPPG, ao curso de pós-graduação ou a uma disciplina do curso de pós-graduação.

 

Fluxo de Eventos:

 

            Fluxo Básico:

 

01- Usuário preenche as informações da atividade e do período;

02-Usuário requisita a confirmação da operação;

03-Sistema confirma a operação, cadastrando a atividade.

 

 

 

Fluxo Alternativo:

 

·        No item 2, o sistema exibe mensagem contendo campos obrigatórios que não foram preenchidos e aqueles contendo informações inconsistentes, o sistema sugere que sejam fornecidas e alteradas, respectivamente.

 

3- Caso de Uso C.L.3 Remover Atividade

ID: C.L.3

 

Este caso de uso realiza a exclusão de uma atividade pertencente a um calendário.

 

Fluxo de Eventos:

 

            Fluxo Básico:

 

01-Usuário seleciona o calendário que contém a atividade que deseja excluir;

02-Sistema lista as atividades do calendário selecionado;

03-Usuário seleciona quais atividades que deseja excluir;

04-Usuário requisita a confirmação da operação;

05-Sistema confirma a operação, removendo a atividade.

 

 

Fluxo Alternativo:

 

·        Não possui Fluxo Alternativo.

4 - Caso de Uso C.L.4 Selecionar Atividade

ID: C.L.4

 

A finalidade deste caso de uso é selecionar uma atividade cadastrada. Para que o sistema possa listar a atividade, deverá ser informado o calendário e outros parâmetros que filtrarão os resultados desta atividade. Tais como: área da atividade, nome da atividade, data, etc.

 

 

Fluxo de Eventos:

 

            Fluxo Básico:

 

01   Usuário informa o calendário e outras informações que desejar;

02   Usuário requisita a confirmação da operação;

03   Sistema confirma a operação listando as Atividades selecionadas.

 

Fluxo Alternativo:

 

 

·        No item 02, se o usuário fornecer alguma informação que não possui nenhuma atividade, sistema apresenta uma mensagem dizendo que não tem atividade.

 

 

5 - Caso de Uso C.L.5 Alterar Calendário

ID: C.L.5

 

A finalidade deste caso de uso é alterar as informações registradas nos calendários participantes do sistema.

 

Fluxo de Eventos:

 

            Fluxo Básico:

 

01    Usuário seleciona Calendário que deseja alterar;

02    Usuário preenche as informações do Calendário que deseja alterar;

03   - Usuário requisita a confirmação da operação;

04   Sistema confirma a operação, alterando o Calendário.

 

Fluxo Alternativo:

 

·        No item 03, o sistema verifica se existem informações inconsistentes, ele retorna uma mensagem dizendo que os dados estão incorretos.

ID: C.L.6

Fluxo de Eventos:

02   - Usuário requisita a confirmação da operação;

03   - Sistema confirma a operação, listando o Calendário.

 

Fluxo Alternativo:

 

·        Não possui Fluxo Alternativo

Fluxo de Eventos:

 

            Fluxo Básico:

8 - Caso de Uso C.L.8 Alterar Atividade

ID: C.L.8

 

A finalidade deste caso de uso é alterar as informações das atividades registradas nos calendários.

 

Fluxo de Eventos:

02   - Sistema lista as Atividades do Calendário selecionado;

03   - Usuário seleciona a Atividade que deseja alterar;

04   - Usuário preenche as informações da Atividade que deseja alterar;

05   - Usuário requisita a confirmação da operação;

06   Sistema confirma a operação, alterando a Atividade.

 

Fluxo Alternativo:

 

·        No item 05, o sistema verifica se existem informações inconsistentes, ele retorna uma mensagem dizendo que os dados estão incorretos.
